# 本机启动 / 重启 Online 客户端

## 仅前端静态（连远程 API，最常见）

- **启动**：在 `lobster_online` 根目录执行  
  `./start_online.sh`  
  浏览器请打开 **`http://127.0.0.1:8000/`**（**不要**带 `?api=`；本机访问时 `static/js/app.js` 已默认连 `http://42.194.209.150`）。端口可用 `ONLINE_CLIENT_PORT` 覆盖。
- **重启（端口被占用时先释放）**：
  ```bash
  # 释放默认端口 8000（必要时强杀）
  for p in $(lsof -ti :8000 2>/dev/null); do kill "$p" 2>/dev/null; done
  sleep 1
  for p in $(lsof -ti :8000 2>/dev/null); do kill -9 "$p" 2>/dev/null; done
  cd /path/to/lobster_online && ./start_online.sh
  ```
  若改了 `ONLINE_CLIENT_PORT`，应对**该端口**做同样的 `lsof` + `kill`。

底层即：`python3 scripts/serve_online_client.py [端口] [API_BASE]`（第二个参数仅用于启动时打印说明；默认 API 以 `app.js` 为准。若要连其它 API，可手动访问一次带 `?api=` 的 URL 或设置 `localStorage.lobster_api_base`）。

**发布管理 / 素材库**：依赖本机 `lobster_online` 后端（`LOCAL_API_BASE`）。仅静态页时页面会提示执行 `LOBSTER_EDITION=online python3 backend/run.py`，与「企微配置数据在本地、权限在 server」一致。

## 需要本地业务能力时（发布 / 素材 / LOCAL_API_BASE）

另起 **lobster_online 后端**（与 Cursor 规则 `auto-update-restart-after-changes.mdc` 一致）：

```bash
cd /path/to/lobster_online
# 改代码后重启示例：
pkill -f "backend/run.py" 2>/dev/null || true
LOBSTER_EDITION=online python3 backend/run.py
```

注意：若后端与静态服务都默认占 **8000**，只能二选一或改其中一方的端口，避免冲突。

## 发布管理「打开浏览器」无窗口（Windows）

- **Chromium**：需已安装 Playwright 使用的 Chromium——完整包带 `browser_chromium/` 时，后端会自动使用该目录；否则在本包根目录执行：  
  `python\python.exe -m playwright install chromium`（或先跑完 `install.bat`）。
- **启动方式**：`run_backend.bat` 已设置与 `start.bat` 相同的 `PLAYWRIGHT_BROWSERS_PATH`；`publisher/browser_pool.py` 也会在未设置环境变量时自动指向包内 `browser_chromium`。
- **无屏 / 另一台电脑访问**：若用 `start_headless.bat`，Chromium 会在**运行后端的那台机器**上弹出，远程浏览器里点「打开浏览器」本机可能**看不到**窗口，需到服务器显示器或 RDP 会话里操作。
- **仍失败**：看 `backend.log`；前端若弹出具体错误文案，多为 Playwright 报错（缺浏览器、权限等）。
