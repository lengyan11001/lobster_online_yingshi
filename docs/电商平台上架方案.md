# 电商平台商品上架方案

## 目标

在现有 Playwright 自动化发布框架上扩展电商商品上架能力，支持：
- 淘宝、拼多多、抖店（抖音电商）、小红书店铺、1688
- **自动填充**商品信息（标题、描述、图片、价格、SKU 等）
- **最后一步不点发布** — 表单填好后将浏览器窗口置前，等用户自己确认并点击发布

## 1. 平台信息

| 平台 | platform_id | 商家后台 URL | 新品入口 |
|------|-------------|-------------|----------|
| 淘宝 | `taobao` | https://sell.taobao.com | 发布宝贝 → 商品发布 |
| 拼多多 | `pinduoduo` | https://mms.pinduoduo.com | 商品管理 → 发布新商品 |
| 抖店 | `douyin_shop` | https://fxg.jinritemai.com | 商品 → 商品创建 |
| 小红书店铺 | `xhs_shop` | https://ark.xiaohongshu.com | 商品 → 发布商品 |
| 1688 | `ali1688` | https://work.1688.com | 商品管理 → 发布商品 |

## 2. 架构设计

### 2.1 复用现有框架

```
backend/app/api/publish.py          → 扩展 SUPPORTED_PLATFORMS，新增电商路由
publisher/browser_pool.py           → 复用 Playwright 持久化上下文 + 代理
publisher/drivers/__init__.py       → 注册新驱动
skills/<platform>_shop_publish/     → 每平台一个驱动包
```

### 2.2 与现有"社媒发布"的关键区别

| 维度 | 社媒发布 | 电商上架 |
|------|---------|---------|
| 内容类型 | 视频/图片 + 标题 + 文案 | 商品信息（多图、SKU、价格、详情页） |
| 自动化程度 | 全自动到发布完成 | **填充到待发布**，不点最终提交 |
| 结束状态 | `success` / `failed` | `draft_ready`（草稿已填好） |
| 浏览器行为 | 可无头 | **必须有头**，最后置前让用户审核 |

商品上架请求结构（新建 Pydantic model）：
```python
class EcommerceListingRequest(BaseModel):
    account_id: int                          # 店铺账号 ID
    title: str                               # 商品标题
    description: Optional[str] = None        # 商品描述/详情
    category: Optional[str] = None           # 类目（平台类目路径或ID）
    price: Optional[str] = None              # 价格
    stock: Optional[int] = None              # 库存
    images: Optional[List[str]] = None       # 商品主图 URL 列表
    detail_images: Optional[List[str]] = None # 详情页图片 URL 列表
    sku_list: Optional[List[dict]] = None    # SKU 变体列表 [{name, price, stock, image_url}]
    shipping_template: Optional[str] = None  # 运费模板名称
    options: Optional[dict] = None           # 平台特有字段
```

### 2.4 Driver 接口

```python
class EcommerceDriver(BaseDriver):
    """电商上架驱动基类"""
    platform_type = "ecommerce"
    
    async def navigate_to_listing(self, page) -> bool:
        """导航到商品发布页"""
        raise NotImplementedError
    
    async def fill_product_info(self, page, listing: dict) -> dict:
        """
        填充商品信息到表单。
        返回 {"ok": True/False, "fields_filled": [...], "fields_skipped": [...]}
        不点击发布按钮。
        """
        raise NotImplementedError
    
    async def bring_to_front(self, page) -> None:
        """将浏览器窗口置前，让用户审核"""
        await page.bring_to_front()
```

### 2.5 上架流程

```
用户/对话 → POST /api/ecommerce/listing
    ↓
1. 验证 account_id → 获取店铺账号 + browser_profile
2. 打开 Playwright 持久化浏览器（有头模式）
3. 检查登录状态 → 未登录则提示扫码
4. 导航到「发布商品」页面
5. 按字段逐一填充（标题 → 类目 → 图片上传 → 价格 → SKU → 详情）
6. ★ 不点击「发布/提交」按钮 ★
7. 将浏览器窗口置前
8. 返回 {"status": "draft_ready", "message": "商品信息已填充，请在浏览器中检查并手动点击发布"}
```

## 4. 各平台难点预估

| 平台 | 难点 | 预估 |
|------|------|------|
| 抖店 | 类目选择器是搜索+树结构；SKU 组合较复杂 | 中等 |
| 淘宝 | 反自动化检测强；图片上传组件复杂 | 高 |
| 拼多多 | 页面结构相对简单 | 低-中 |
| 1688 | 与淘宝同体系但面向批发，字段较多 | 中等 |
| 小红书店铺 | 商家后台更新频繁，选择器容易变 | 中等 |

## 5. 注意事项

1. **绝不自动点发布** — `draft_ready` 状态后将窗口置前，剩余操作完全交给用户
2. **登录靠用户扫码** — 电商后台安全要求高，不做自动登录
3. **每平台独立 profile** — 与社媒账号同样，每个店铺独立浏览器数据目录
4. **图片需公网可访问** — 上架时从素材库下载到本地再上传到平台
5. **代理支持** — 复用现有 `meta.browser.proxy_*` 字段
